# 셔클 지역별 모니터링 리포트 자동생성 툴 개요

## 1. 문제의식

<img src="shucle_insight_main.png" alt="셔클 인사이트 대시보드 메인화면" height="200" style="display:block; margin: 0 auto;">

- **인사이트 툴 기반 이슈 발굴 한계** — 약 130개 차트 수동 조회 필요, 지표 간 연관관계 종합 판단 어려움, 이슈 사후 인지
- **지역/기간별 데이터 수집 시 장시간 소요** — 6개 탭 순회 반복 작업, 복수 지역에 대한 반복 작업 필요
- **지자체 공유용 주간/월간 모니터링 리포트 수요** — 매 주기 데이터 수동 취합 및 양식 정리에 추가 공수 발생
- **이해관계자 간 효율적 현황 공유 필요** — 지자체/플랫폼사/운영사가 동일 분석 기준으로 소통 가능한 정형 리포트 부재

---

## 2. 작동 프로세스

```
[수동 로그인] → [자동 데이터 수집] → [자동 리포트 생성] → [산출물(HTML/XLSX)]
```

1. **수동 로그인** — 인사이트 통계 툴 브라우저 수동 로그인(문자인증), 세션은 브라우저 프로필에 저장하여 이후 재활용
2. **자동 데이터 수집** — 스크립트 실행 시 지역/기간 입력, 브라우저 자동화(Playwright)로 6개 탭 차트 데이터 API 응답(약 126개) 자동 캡처, JSON 저장, 데이터 검증 수행
3. **자동 리포트 생성** — 데이터 분석 체계에 따라 분석기간/비교기간 1차 핵심지표 산출, ±10% 초과 변동 시 2차 세부지표 분석 수행, HTML/XLSX 형식 리포트 자동 생성

---

## 3. 데이터 분석 체계

4개 카테고리 15개 1차 핵심지표 기반 운영 상태 진단, ±10% 이상 변동 시 2차 세부지표로 원인 추적

### 1차 핵심지표

| 카테고리 | 핵심 질문 | 지표 |
|----------|-----------|------|
| **수요** | 얼마나 많이 이용하는가 | 실시간 호출 건수, 이동완료 호출 건수, 총 탑승객 수, 대당 탑승객 수 |
| **품질** | 서비스 경험이 좋은가 | 평균 대기시간, 평균 우회비율, 평균 이동시간 |
| **공급** | 차량이 충분히 운영되는가 | 운행차량 대수, 대당 운행시간, 가호출 성공률, 대당 운행거리, 경로이탈비중 |
| **성장** | 이용자가 늘고 있는가 | DAU, 신규 지역 회원, 누적 지역 회원 |

### 2차 세부지표 드릴다운

1차 지표 비교기간 대비 ±10% 이상 변동 시, 연결된 2차 세부지표 자동 분석

> 예) 평균 대기시간 +10% 증가 → 상위 10% 대기시간, 장시간 대기(30분+) 비율, 운행차량 대수, 호출 건수

### 리포트 구성

1. **1차 핵심지표 요약** — 15개 지표의 비교기간/분석기간 값, 변동률, 상태
2. **2차 세부지표 (변동)** — ±10% 초과 변동 지표의 원인 분석 드릴다운
3. **2차 세부지표 (안정)** — ±10% 이내 안정 지표의 세부 현황
4. **핵심 해석** — 데이터 기반 종합 해석

---

## 4. 활용 방식

- **이슈 조기 발견** — ±10% 이상 변동 지표 자동 감지, 연관 지표 드릴다운으로 원인 파악 지원
- **지역 간 비교** — 동일 분석 체계 기반 복수 지역 리포트 생성, 운영 수준 비교 용이
- **정형 리포트 공유** — HTML/XLSX 산출, 별도 가공 없이 지자체 보고 및 내부 공유 활용 가능

---

## 5. 한계

- **DB 직접 접근 불가** — 브라우저 자동화로 대시보드 API 응답 캡처 방식 사용, DB 직접 쿼리 대비 속도/유연성 제약
- **수집 시간 소요** — 지역당 약 5분, 비교기간 포함 시 약 10분, 다수 지역 시 선형 증가
- **데이터 누락 가능성** — lazy-loading 타이밍/네트워크 상태에 따른 일부 누락 우려, 수집 완전성 검증(126개 차트 대비 수집 수 비교) 및 zone_id 지역 검증으로 보완
